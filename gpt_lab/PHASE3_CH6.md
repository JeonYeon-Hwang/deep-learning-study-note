## Fine Tune  
  
튜닝이란?.. pre-trained(이미 학습한) 모델을 가져와, 특정 용도로 조정(?)하는 것  
예: 기존 GPT 모델(다음 올 단어를 학습함) → 번역기로 튜닝(원어=번역어 k-v 구조로 변환)  
  
*그렇다면 make_sentiment_dataset은 어떤 역할을 하나?*  
raw 데이터 가공하여 dataset을 만듦: `.tsv` → `train/validation/test`  
  
*예시를 보자면..*  
NSMC원본(.tsv)파일.. 리뷰가 긍정/부정인이 맞추는 샘플:  
```
id  document          label
90  올해 본 최고의 영화  1
98  액션도 별로 개연성 꽝   0
```
이것이 dataset으로 만들어 진다면 다음과 같은 형태가 된다:  
```
train_data = [
    {"text" : "올해 본 최고의 영화", "label": 1},
    {"text" : "액션도 별로 개연성 꽝", "label": 0}
]
```
데이터는 train용, validation용, test용 세 가지를 필요로 한다.  
  
*추가 요소도 지속적으로 연산된다*  
단어 예측하는 연산 로직과 해당 전체 문장의 긍정/부정 뉘앙스 파악 로직은 동일하다  
예시: 액션도 → 별로 → 개연성 → 꽝.. 예측하면서, attention에서 업데이트 지속  
최종 출력층에서는 긍정/부정 최종 결론을 내림
