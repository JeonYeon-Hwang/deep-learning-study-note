## ch. 5 중심으로 학습하기  
책의 ch. 5의 초창기 부분에 있다. 계산 그래프로.. 각 노드를 기준으로 순차적 연산 진행.  
이 계산 그래프를 **역순**.. 즉, "역전파"로 바꾸면, 각 단계에서 특정 정보값을 추출할 수 있다. 

*좀 중요한 개념: 연쇄법칙*
일단, 연쇄법칙은 그래프 상의 역전파와 같다고 함.. 예시를 들자면:  
z = (x + y)^2 라는 합성함수는.. 
```
1단계: z = t^2  
2단계: t = x + y 
```
로 나눠진다. 그리고... 각 단계에는 역으로 가기 위한 "미분값"이 존재하며,  
각 미분값을 적용하면(곱하면) 전 단계의 값이 추정되며 이를 **역전파**라고 한다.  
또한, 동시에 "각 단계의 미분값을 연속적으로 곱하면" 바로 몇 단계 점핑이 가능하다.  
→ 이를 연쇄법칙이라고 한다!  

*활성화 함수란?*  
여러 inputs를 받은 한 노드가 출력하기 위한 계산 요소 중 하나이다. 그렇다면 어떤 요소?  
노드 내부에서 연산되는 과정을 요약하면 다음과 같다:  
.. 1단계. 단순 합산: y = (입력1×가중치1 + 입력2×가중치2..) + b  
.. 2단계. 활성화: 결과값 = 활성함수f(y)  
여기서 활성화라는 말이 붙는 이유는, 출력 신호 전달 유무를 계산하는 최종 필터이기 때문.  
<br>

**연산 계층**
하나의 노드의 흐름을 도식화 하자면 다음과 같다:  
```
[입력 x] ➔ [가중치 w] ➔ (곱셈 계층) ➔ [결과 xw] ➔ (덧셈 계층) ➔ [최종 점수 y]
                                                   ↑
                                               [편향 b]
```
<br>

*연산 단위(?)는 계층으로 표현이 가능하며, 이를 3단계로 분류가 가능하다*
→ 예시는 픽셀로 이루어진 이미지를 판단하는 것을 예시로 듦.  

**[입력 데이터: 784px image]**  
↓  
**여기는 단일 노드에 존재**
1 구간(Affine 레이어): 순수 수학 연산 담당 .. 예시: $y = xw + b$
2 구간(Activation 레이어): ReLU/Sigmoid 사용(복합연산) .. 예시: $f(x) = \max(0, x)$
↓  
**3 구간(Softmax 레이어)**: 정답과 비교해서 "오차 성적표" 산출, 역전파로 재귀적? 수정

### ReLU 구현하기
여기서 의문.. "왜 ReLU 먼저 구현을 하라고 했을까?" → 답은 비교적 허무했다: 비교적 쉬워서  

**ReLU 코드 작성하기**
원 단어는 Rectified Linear Unit이다. 논리는 비교적 간단하다: 양수는 양수, 음수는 0처리  
이를 python의 numpy 배열로 구현하여서 비교적 난해한데, 이를 정리하면 다음과 같다:  
1. 순전파 작성하기  
   self.mask = (0 이하면 True로 마킹하기)  
   out = 원본배열.copy() .. 만약 배열 [2, -1, 3]이 있다면 out에 그대로 복사  
   out[배열 중 mask에서 True인 요소는] = 0으로 값 변환 .. 배열은 [2, 0, 3]이 된다. 
<br> 

2. 역전파 작성하기  
   이미 앞 노드에서의 연산한 미분값 배열 dout가 온다.  
   연쇄법칙에 따라, 현 ReLU의 미분값(양수면 1, 음수면 0)을 전달하면 된다.  
   dout[기존 미분값들 중 mask 가 True인 요소는] = 0으로 값 변환한다(1은 곱해도 그만)  
   이를 dx 새 배열에 할당하여 반환한다.  
<br>

*왜 forward 테스트는 2가지로 더 나눴나?*  
테스트 시나리오 목록은 아래와 같음.. 양수 혹은 음수 특정 부분에서 미구현 감지 목적이라고


### Softmax 구현하기
softmax는 왜 필요한가?  
→ "추론"할 때에는 사실 이 계층을 필요하지 않다: Affine계층의 가장 높은 점수가 답이기 떄문  
→ "학습"을 하려면 필수인 계층이다: 점수를 정규화해 원답과 비교하여 역전파를 해야하기에..  

**구현 개요**  
구현 방식은 비교적 간단하다.. 식은 이렇게: 개별 요소 확률 = 개별 요소 점수 / 전체 점수 합  
그러나!! 이것은 softmax_forward.. 즉, 정방향에서만 유효한 계산법이다.  

*softmax_backward는 무엇이 다른가?*  
교재에 나오는 "softmax-with-loss"와 사실상 동일.. 자세한 내용은 추후에   
<br>

**Softmax 코드 구현하기(ch. 3 참고)**  
$e^x$를 사용하는 이유: 각 요소의 점수를 다 더하는 방식은 점수가 "음수"이면 꼬이게 된다  
여기에 ${e^{x + \log C}}$ 하는 이유: e의 경우 너무 커져 overflow가 되는 것을 방지  

softmax_forward 구현 요약:  
가장 큰 수 = np.max(배열 x) .. np(numpy)를 사용하여 가장 큰 수 뽑기  
$e^x$화된 배열 = np.exp(배열 x - 가장 큰 수) .. 이것으로 새 배열을 만들 수 있다  
모두 더한 값 = np.sum($e^x$화된 배열) .. 이를 모두 더하고  
정규화 배열 = $e^x$화된 배열 / 모두 더한 값 .. 정규화된 배열을 뽑아내 반환  
주의!!  `axis=-1, keepdims=True`를 추가하여 각 행마다 독립적으로 수행해야 함  

softmax_backward 구현 요약:  
train()함수에서 다 하도록 프로젝트에서는 구현이 되어 있음  
train에선 뭘 하는가? → 딥러닝 모델의 오케스트레이션 역할을 함.. 순서는 다음과 같다:  
정답과 비교(오차 점수 산출) → 오차 정산(backward 진행) → 반영하여 가중치(weights) 수정  
<br>

### Affine 구현하기
Affine을 하려면 최소 두 가지 요소(W, B)를 필요로 한다.. 이를 식으로 표현하면: 
$Y = X \cdot W + B$ 와 같이 된다.. W: weight, B: bias, X: input  

*상세히 설명을 한다면?*  
$X \cdot W$ 은 **선형변환(Linear)** 의 일종이다. 여기에 $+ B$ 를 하면 **Affine변환**이다.  

affine_forward 구현 요약:  
결과값 = dot(행렬 곱을 수행하는 메서드) + 가중치  

affien_backward 구현 요약:  
다음과 같은 변화량(미분값)을 구해야 한다.. dw = W곱한 변화량, db = B를 더한 변화량  
너무 난해하다.. 일단 구현 식을 표현하고 이해한 것을 기술해보자:  

$\frac{\partial L}{\partial W}$ = np.dot(self.x.T, dout) .. $dw$를 나타냄 
→ 행렬 x의 역에 dout를 곱하면 dw가 나온다.  

$\frac{\partial L}{\partial B}$ = np.sum(dout, axis=0) .. $db$를 나타냄  
→ 더하는 요소라 좀 다르다: 더하기 미분은 그냥 1이기 때문..  
→ 왜 그런 지 모르겠지만(axis=0).. 즉, 세로로 요소의 합을 한 것이 db라고 함.  

$\frac{\partial L}{\partial X}$ = np.dot(dout, self.W.T) .. $dx$를 나타냄  
→ 여기서 .T는 행렬 W를 뒤집는다는 것을 의미.  
→ dout에 가중치 행렬 W를 뒤집어 곱하면 dx가 나온다.  
<br>

### cross_entropy_loss 구현하기  
역전파의 dout가 최초로 생성되어 출발하는 **시작점**이다.  
모델이 예측한 값(`y_pred`)와 실제 정답(`y_true`)와 비교를 통해 **최종 오차 점수(L)**을 구함.  
계층의 흐름을 도식화하면 다음과 같다:  
```
[입력층]               [은닉층 반복]       [출력층]       [손실 함수]
 픽셀 값 🡢 [Affine] 🡢 [ReLU] 🡢 ... 🡢 [Softmax] 🡢 [Cross Entropy]
  (X)                                   (y_pred)          │
                                                          ▼
[입력층]               [은닉층 반복]                 정답지(y_true)와 비교    
  [dx]  🡠 [Backward] 🡠 [Backward] 🡠 [Backward] 🡠   [최초 dout]      
```
<br>

**오차 점수(L) 구하기**  
여기서는 오차 점수를 구하고 반환하는 계층이다. 순서를 말하자면 다음과 같다:  
1. 확률 행렬(y_pred: 각 행이 하나의 답)과 정답 레이블(y_true: 하나가 한 행의 답 index)  
2. 정답 위치만 골라낸 다음(np.arange 사용) → 새 배열로 만들어 내기  
3. 안전장지(np.clip 걸기).. 왜? → 얘를 log화 할 것인데, 0의 경우 -\(\infty \)가 되기 때문  
4. 로그에 집어넣어 점수 변환하기: 1이하이므로 변환된 점수는 무조건 음수  
5. 이들을 합산 및 양수화, 그리고 bach size 만큼 나눈 것이 → 오차 점수가 된다  
<br>

**코드로 구현하기**  
```
batch_size = y_pred.shape[0] => 전체 행의 갯수를 의미

answer_probs = y_pred[행 좌표, 열 좌표] 
=> 행: np.arange로 0, 1, 2 .. size만큼  
=> 열: y_true를 기준으로 함

log_probs = np.log(np.clip(answer_probs, 1e-7, 1.0))
=> log 하면 바로 변환되나, clip로 한계 설정  
=> 한계선은 1e-7에서.. 1.0 사이까지

sum_probs = log_probs의 값을 모두 더하고
return sum_probs을 batch_size 만큼 뺀다
```
