딥 러닝의 3단계 순환 주기:
1단계(정방향): 처음 → 끝 .. 문제를 받아 통과시키며 답을 뱉는다  
2단계(역방향): 처음 ← 끝 .. 오차 점수에서 출발, 오차 기여도(Gradient)를 계산  
3단계(역방향): 처음 ← 끝 .. 기여도를 중심으로 **SGD**가 가중치를 깎고 고쳐버림

**3단계: 매개변수 갱신**  
가중치(w) 및 바이어스(b)는 초기에는 모두 무작위로 설정되어 있다  
초기설정: 784개의 input, 10개의 output, 13,002개의 w와 b(파라미터)  

*SGD에 대해 알아보자*  
확률적 경사 하강법(SGD)는.. Cost라는 함수의 특정 위치에서 가장 낮은 곳 찾기이다  
구현 코드는 다음과 같다:
```
def update(self, 파라미터, 오차기여도)
    for key in 파라미터.keys():
        파라미터[key] -= self.lr * 오차기여도[key]
```
이는: `새로운 가중치 = 기존 가중치 - (학습률 × 오차기여도)`를 구현한 코드이다  
→ `파라미터`와 `오차기여도`는 key를 공유하는 쌍둥이 dictionary이다  
→ `lr`는 공용 보폭(?)으로, 개별 지도에 어느정도 공통으로 조정할 지를 결정함  

SGD의 단점? .. 비등방성(하나로 움푹 페이지 않은)함수에서는 탐색이 비효율적!  
<br>

*Adam에 대해 알아보자*  
SGD(길치 + 보폭 조절 못함)의 업그레이드 판.  
Adam은 가중치 조절 시, 과거의 기록을 저장 및 이에 기반하여 갱신을 한다.  
Momentum: 원래 가던 방향에 관성이라는 속성을 추가  
AdaGrad: `lr`공용 보폭을 각 개별 노드별로 조절 → 많이 수정된 노드는 적게 조절  
<br>

*Adam 구현하기*  
Momentum과 AdaGrad 코드가 이미 있다. 적당히 짬뽕?시켜야 하는데, 공식은 이렇다:
`params[key] -= (보정된 v) / (np.sqrt(보정된 h) + 1e-7) * 공용 lr`

이를 구현하는 부분은 이렇게 하였다:
```
과거 방향 기록 9할 남기고 현재 기울기의 1할만 더해 갱신
self.v[key] = 0.9 * self.v[key] + 0.1 * grads[key]

과거 보폭의 99.9% 남김 + 현재 보폭의 0.1%만 적용해 갱신
self.h[key] = 0.999 * self.h[key] + 0.001 * (grads[key] ** 2)

보정 효과: 첫 t에서는 크고 이후에는 작아져 1에 수렴
v_corrected = self.v[key] / (1.0 - 0.9 ** self.t)
h_corrected = self.h[key] / (1.0 - 0.999 ** self.t)           

보정된 속도를 "보정된 보폭의 루트 값"으로 나눔, 기본 학습률을 곱해서 가중치 깎기
params[key] -= self.lr * v_corrected / (np.sqrt(h_corrected) + 1e-7)

```
