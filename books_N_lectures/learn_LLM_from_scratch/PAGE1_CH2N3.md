## Ch. 2 정리

임베딩: 특정 데이터를 벡터로 표현하는 것  
비슷한 단어 = 비슷한 공간 상의 백터를 가짐   
토큰화.. 특정 단어: 특정 ID로 할당한 dict을 만드는 것을 의미  
특수토큰: 개발자가 셋팅을 한 비단어 토큰 → 예: 문장의 끝 표현 토큰  

*슬라이딩 윈도우란?*  
단어 대 단어, 1 대 1 페어 뿐만 아니라, 윈도우(복수의 단어들) 기준으로,  
N개 대 N개로 페어 설정이 가능하다.  

만약 10개의 문장의 페어 경우의 수? → 어마하게 많아지며, 이들을 ID로 저장  
이 페어들은 모델 구축에 사용된 후 사라지는 **휘발성 데이터**이다.  

*토큰 임베딩이란?*  
각 단어(토큰)을 N개의 차원을 가지는 "하나의 벡터"로 표현하는 것: 초기에는 랜덤  

위치 임베딩은 왜 필요한가? .. 슬라이딩 윈도우의 페어는 "순서"를 보장하지 않음.  
컴퓨터는 순차적으로 읽지 않기 때문에, 각 단어의 순서를 담은 정보가 필요. 
한 단어 토큰 임베딩 = 의미 벡터 + 위치 벡터 .. 합쳐져서 사용함

## Ch. 3 정리
  
트랜스포머: 다음 단어 예측하는 모델(확률적 추측)  
각 단어(토큰)는 다차원 백터값들을 가진다 → 어텐션 블록을 통과하면서 값 업데이트 진행
.. 문맥을 파악한다: 다른 단어와의 관계 파악함.  
단어의 백터값 예시: $queen = king + woman - man$  
또 다른 예시: $ \overrightarrow{Plur}  := E(cats) - E(cat)$  
결국 다음 단어를 찾는 것은.. 앞의 맥락(복수의 단어들)을 파악하여 맞추는 것  
Temp 설정..  낮을 수록 집중도가 올라감  

어텐션 메커니즘(attention mechanism) .. 트랜스포머 구조  
기존의 학습된 단어 백터는 "단어 사전"으로서 고정값을 보유  
문장 출현 → 문장 내 단어들의 관계를 계산하여 동적인 벡터값으로 업데이트 함  

표로 정리한 다차원 백터 종류:  
| 이름 | 역할 |
| :--- | :--- |
| Query | 본인과 연관된 단어가 무엇인지 질문함 |  
| Key | 해당 단어와 자신이 연관 있음을 매칭해줌 |
| Value | 단어가 내포한 실제 의미를 가진 정보 공간을 가리킴 | 

<br>

*Attention Pattern* 
Q와 K 모두 다차원 백터값 형태이나, dot 연산을 통해 하나의 각 값으로 산출됨  
Q x K = 관계있을 수록 내접값이 큰 양수가 나옴(열 기준 softmax로 확률분포도 산출 可)  
Q x K 에서 뒷 단어가 앞에 오는 경우의 분포표는 무시(혹은 0으로 처리) → **마스킹**의 형태   
V백터는 여러 K와 상호작용한 요소를 기준으로 산출된다
